18 July 2022 • Niels Ophey • 1 min read

Namenskonvention

Es ist immer eine große Diskussion, wie eine Namenskonvention für die bereitzustellenden Azure-Ressourcen implementiert werden kann. Oft wird dieser Artikel aus dem Microsoft Cloud Adoption Framework for Azure als Ausgangspunkt für die eigene Implementierung in einem Unternehmen verwendet.

Dieses Diagramm zeigt die Komponenten eines Azure-Ressourcennamens:

Nächster Schritt

Was aber, wenn Sie die Namenskonvention Ihrer Umgebung in großem Maßstab und während der Bereitstellung mit Pipelines verwalten möchten? Es gibt ein hilfreiches Tool, das Sie in Betracht ziehen sollten - das Azure Naming Tool!

Das Azure Naming Tool wurde unter Verwendung eines Benennungsmusters entwickelt, das auf Microsoft’s best practices basiert. Nachdem die Organisationskomponenten von einem Administrator definiert wurden, können Benutzer das Tool verwenden, um einen Namen für die gewünschte Azure-Ressource zu generieren.

Das Interessanteste daran ist, dass Sie dieses Tool als API bereitstellen können, die aus Ihrer Automatisierung heraus verwendet werden kann.
